# Skill Quality Standard

Every active skill must contain frontmatter, output marker, overview, workflow or rules,
quality gates when judgment is involved, examples when useful, hard stops, usage
checklist, and cross-references.

Active routed modules need direct routing evidence in metadata and concrete
operating language in instructions. Strong signals describe request or repository
evidence directly; they do not restate the module name or use generated phrases
such as "evidence for <module>". Quality gates, hard stops, and checklists must
name the module's real responsibility, risk, and validation path rather than
title-swapped boilerplate. Prefer progressive disclosure and validation over
broad doctrine. Large stable details belong in workspace references. Examples
use Candidate, JobOffer, Application, Recruiter, Company, Interview, Contract,
and SkillTag.
