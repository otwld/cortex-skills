# Skill Quality Standard

Every active skill must contain frontmatter, output marker, overview, workflow or rules,
quality gates when judgment is involved, examples when useful, hard stops, usage
checklist, and cross-references. Prefer concrete operating language, progressive
disclosure, and validation over broad doctrine. Large stable details belong in workspace
references. Examples use Candidate, JobOffer, Application, Recruiter, Company,
Interview, Contract, and SkillTag.
